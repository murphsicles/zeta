# v0.3.22 Blockers

## Remaining v0.5.0 Compilation Issues

Based on verification of v0.3.21, the following blockers remain for full v0.5.0 compilation:

### 1. Missing Stub Implementations for Code Generation
- **Issue**: Virtual modules provide type definitions but not implementations
- **Examples**: `is_cache_safe`, `lookup_specialization`, `parse_zeta` functions
- **Impact**: Code compiles (parses + type checks) but fails during codegen
- **Solution**: Implement minimal stub functions in virtual module system or link to actual implementations

### 2. UTF-8 BOM Handling in Parser
- **Issue**: Parser fails on files with UTF-8 BOM
- **Impact**: Test files with BOMs fail to parse
- **Solution**: Strip BOM or handle it in parser

### 3. Stack Overflow in Complex Parsing
- **Issue**: Stack overflow when parsing complex v0.5.0 code with multiple imports
- **Impact**: Some files cause stack overflow (though parser recovers)
- **Solution**: Optimize parser recursion or increase stack size

### 4. Missing Runtime Implementations
- **Issue**: `println!` macro and other Rust-style constructs not supported
- **Impact**: Code with `println!` compiles but macro isn't expanded
- **Solution**: Implement basic macro expansion or provide Zeta alternatives

### 5. Incomplete Virtual Module Coverage
- **Issue**: Not all `zeta::` module paths have virtual module implementations
- **Impact**: Some v0.5.0 imports may fail
- **Solution**: Expand virtual module system to cover all needed paths

## What's Needed for v0.3.22
