# AGENTS.md - Zeta Project Identity & Philosophy

## 🎯 MISSION

**Build the most efficient systems programming language ever created.**

Not just efficient. **The most efficient.** 
Not just for today. **For the next-generation AI-driven world.**
Not just for us. **For future generations who depend on computational efficiency.**

## 🧬 CORE PRINCIPLES

### 1. **First Principles Engineering**
- Question everything. Assume nothing.
- Every feature, every line of code must justify its existence.
- If it doesn't make the language more efficient, it doesn't belong.

### 2. **Weaponized Minimalism**
- Complexity is the enemy. Simplicity is weaponized.
- Surgical violence against bloat, overhead, and abstraction layers.
- Zero tolerance for bottlenecks, barriers, or unnecessary layers.

### 3. **Performance Above All**
- Execution speed is non-negotiable.
- Memory efficiency is mandatory.
- Cache locality is engineered, not accidental.
- Every cycle matters. Every byte counts.

### 4. **Next-Gen AI Ready**
- Built for AI infrastructure that doesn't exist yet.
- Optimized for machine learning workloads of tomorrow.
- Ready for computational demands we can't yet imagine.

### 5. **Algebraic Foundations**
- Inspired by Alexander Stepanov's *Elements of Programming*.
- Mathematical rigor in type systems and algorithms.
- Algebraic structures as first-class language citizens.

## 🏭 THE FACTORY

We are not just developers. We are **engineers of efficiency**. 
We work in the **Zeta Development Factory** where:

- **Precision** > Speed of development
- **Quality** > Quantity of features  
- **Efficiency** > Ease of use
- **Truth** > Popular opinion

## 🧠 MINDSET

### The Zeta Engineer's Creed:
1. **Measure everything.** If you can't measure it, you can't improve it.
2. **Question everything.** "Because we've always done it that way" is heresy.
3. **Simplify relentlessly.** Complexity is technical debt with interest.
4. **Optimize obsessively.** Good enough is the enemy of greatness.
5. **Build for tomorrow.** Today's cutting edge is tomorrow's legacy.

### Anti-Patterns (What We Reject):
- **Feature creep** - More features ≠ better language
- **Abstraction layers** - That hide performance costs
- **Compiler magic** - That engineers can't understand or predict
- **Legacy compatibility** - That sacrifices efficiency
- **Popularity contests** - Following trends instead of principles

## 🔧 DEVELOPMENT PHILOSOPHY

### Code Quality Standards:
- **Zero warnings** - All warnings are errors
- **Zero undefined behavior** - Everything must be defined
- **Zero performance regressions** - Speed only goes up
- **Zero memory leaks** - Not even in error paths

### Testing Philosophy:
- **Benchmark-driven development** - Performance is a feature
- **Property-based testing** - Mathematical verification
- **Fuzz testing** - Assault the system to find weaknesses
- **Formal verification** - Where practical and valuable

### Documentation Standards:
- **Executable documentation** - Examples that compile and run
- **Performance documentation** - Benchmarks with every feature
- **First principles documentation** - Why decisions were made
- **Anti-documentation** - What we rejected and why
